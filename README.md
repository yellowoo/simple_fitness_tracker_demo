# Align · 左侧力线康复打卡 App

基于你的《左侧骨盆三维扭转康复训练完整计划》制作的 PWA。

## v1.3 修复与重构

### 问题 1:版本更新后数据会保留吗?

**会的,基本无需担心。** 数据存在 iPhone Safari 的 localStorage 里,跟**域名**绑定。只要你用的是同一个部署地址(比如同一个 GitHub Pages URL),即使 `index.html` 换了新版本,打卡历史、感受日记、动作编辑都在原位。

**会丢数据的情况**:
- 主动清 Safari 网站数据
- 删除主屏幕图标 + 90 天内不访问(iOS 自动清理)
- 换了域名(从 GitHub Pages 迁到 Vercel 等)

**保险做法**:每周从"我的"页导出一次 JSON 备份到 iCloud Drive / 相册。

### 问题 2 修复:界面不刷新

昨天打卡后第二天界面没有自动更新到新一天 —— 这是因为 PWA 打开时是恢复了之前的内存状态,不是重新加载。已加入:

- **每分钟自动检查日期变化**,日期变了就刷新整个界面
- **从后台回到前台时自动检查**(visibilitychange 监听)
- **从 iOS 的 back-forward 缓存恢复时刷新**(pageshow 监听)
- **Service Worker 更新策略改为 network-first**:发新版时会弹窗询问"App 有新版本,是否立即刷新?"

### 问题 3 修复:90-90 呼吸计时器无法启动

这是动作类型识别的 bug。现在重写了动作分类系统,**所有带时长的动作都会显示计时器**,详见下方。

### 问题 4 修复:底部按钮遮挡警示内容

- 详情页底部内容区的 padding 从 100px 增加到 140px
- 底部按钮背景从"70% 不透明硬截断"改为"40% 不透明 + 70% 半透明 + 透明"三段渐变
- 加上 backdrop blur(iOS 原生毛玻璃效果)

现在看内容不会被挡住,同时按钮区依然足够清晰。

### 重构:动作分类系统

按你提的三分类,统一了所有判定逻辑:

| 类型 | 识别规则 | 今日页按钮 | 详情页计时器 | 举例 |
|---|---|---|---|---|
| **reps_sets**<br/>次数 × 组数 | "12 次 × 3 组" 类 | 进度按钮 `0/3` | 无 | 死虫、蚌式、鸟狗、分腿蹲 |
| **time_sets**<br/>时长 × 组数 | "90 秒 × 2 组" 类 | 进度按钮 `0/3` + **每组独立计时** | **每组都有** | 侧桥、髋屈肌拉伸、靠墙单腿站 |
| **time_only**<br/>纯时长 | 只有"3 分钟"无组数 | 普通勾选 | 有 | 所有松解动作、90-90 呼吸 |

**time_sets 的新交互**:
- 计时器标题显示"第 1/3 组"等当前进度
- 计时器结束 → 弹闹钟"第 1 组完成" → 关闭闹钟后进度按钮自动变 1/3
- 完成最后一组 → 闹钟显示"全部完成" → 关闭后自动弹质量分(训练类)

## ⚠️ iOS 闹钟使用注意

- **必须保持前台**:iOS PWA 锁屏后 Web Audio 会停 → 默认开启屏幕常亮
- **首次需用户交互**:点"开始"按钮 = 用户交互 = 才能播声音
- **静音键不影响**:Web Audio 不受静音键影响
- **添加到主屏幕后效果最佳**

## 文件清单

- `index.html` — 主应用(~108 KB)
- `manifest.json` — PWA 配置
- `sw.js` — Service Worker(network-first 策略,保证更新能看到)
- `icon-180.png` / `icon-192.png` / `icon-512.png` — 图标

## 部署

GitHub Pages(推荐):
1. 替换仓库根目录的 `index.html` 和 `sw.js` **(这次 sw.js 也要更新!)**
2. 等 1-2 分钟 GitHub Pages 构建完成
3. 在 iPhone PWA 里 —— 如果有"App 有新版本"弹窗就点是;否则**关闭 PWA 重新打开**即可

> 如果更新后还是老界面,可以在"我的" Safari 设置里清掉该域名的网站数据(会丢数据,请先导出备份!),或者在 PWA 下拉菜单里找到"从网页刷新"选项。
